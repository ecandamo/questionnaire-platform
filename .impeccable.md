# Impeccable Design Context

## Design Context

### Product
Internal sales questionnaire platform for a software/API-product company.
Two distinct user surfaces:
1. **Internal dashboard** — mixed internal audience (AEs, presales engineers, CS consultants) creating, publishing, and reviewing questionnaires for prospects
2. **Respondent form** — external audience (prospect IT contacts, business executives, procurement teams) filling out questionnaires via tokenized link, no login

### Users

**Internal (authenticated):**
- Sales AEs, presales/solutions engineers, customer success consultants
- Work context: desk, daytime, often switching between this and other tools (CRM, email, Slack)
- Job: build a questionnaire from a template or question bank → publish → track responses → export/review

**External (respondents):**
- Mixed: could be a CTO, a procurement manager, or an IT architect at a prospect company
- Context: may open the link on any device, at any time, potentially sharing it across their team (collaborative answering)
- Job: complete a structured set of questions, possibly with teammates, then submit

### Brand
- **Company**: API — software/API-product company
- **Brand colors** (FIXED, do not change):
  - API Navy: `#273B6E` → `oklch(0.30 0.10 264)` — primary, sidebar, authority
  - API Green: `#78BC43` → `oklch(0.70 0.17 132)` — active, success, accent
- **Current fonts** (live in `src/app/layout.tsx` via `next/font`):
  - IBM Plex Sans (headings) — `--font-heading` — technical / enterprise tone
  - Source Sans 3 (body) — `--font-sans` — legibility at table density

### Brand Personality
**Modern.** But specifically: precision tooling for professionals — not generic modern SaaS. 

The feel should be: a highly-capable instrument that confident B2B sales professionals reach for. Think Linear.app's density and clarity applied to enterprise sales workflows. Serious without being stuffy. Efficient without being cold.

### Anti-References
- NOT: generic SaaS (Notion-style soft pastel, rounded "friendly" corners everywhere)
- NOT: govtech / legacy enterprise (drab tables, zero whitespace, 1998 vibes)
- NOT: marketing-heavy / landing page aesthetic in a dashboard context

### Aesthetic Direction
**Calibrated authority.** The interface earns trust through precision and restraint, not decoration.

- Dark navy sidebar is the anchor of authority — keep it
- Light content surfaces, not dark-mode dashboard (users and respondents are in daytime/office context)
- Data-dense but never cluttered — information hierarchy through contrast, not borders
- Typography should feel editorial and structured, not playful or generic
- Motion: purposeful only — state transitions, progress feedback, no gratuitous animation
- The respondent form should feel like a premium onboarding experience at a well-run company — trustworthy, unhurried, clear

### Design Principles

1. **Authority through precision** — Every element earns its position. Nothing decorative that doesn't also communicate. Strong typographic hierarchy over border-heavy layouts.

2. **Two distinct tones, one system** — Internal dashboard: dense, efficient, professional tool. Respondent form: calm, guided, frictionless. Same brand DNA, adapted for context.

3. **Restraint as sophistication** — Use the green accent for moments that matter (active states, progress, CTAs). The navy is structural. White space is not filler — it's signal.

4. **Trust as design goal** — The external respondent needs to feel they're in capable hands. Professional typography, clear progress, no noise. Nothing that reads as "Google Form."

5. **Font differentiation matters** — Headings and body use a deliberate B2B pairing (IBM Plex Sans + Source Sans 3), not generic defaults.

### Typography Notes
IBM Plex Sans carries structure and authority in headings; Source Sans 3 keeps long-form and dense UI readable. Adjust scale in `globals.css` / Tailwind if hierarchy needs tuning after font change.

### Theme
Light mode (primary), dark navy sidebar. This is correct for the audience context — sales professionals and business clients use this in office/daytime settings. Dark mode support exists in CSS but is not the priority surface.
